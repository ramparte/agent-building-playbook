## What this project is
A curated, growing collection of *patterns* for building better in the agentic age —
"Programming Pearls, but actionable, for the cognitive-architecture era." One canonical
markdown source → two forms: a dense agent-facing `INDEX.md`, and a human-browsable wiki.
Holistic ("how to build better," not narrowly "build agents"), pattern-level, no provenance
elevation. Contribution via GitHub PRs.

## Curation model (for the offsite)
GitHub PRs are the rail. Maintainer triage = point an agent at the open PRs:
"which are new / where each fits in the tag hierarchy / 2-sentence summary each /
is it redundant, does it extend an existing pattern, or is it a secondary implementation."
No custom triage tooling in v1.

## Firm decisions
- Multi-value dimension TAGS (a pattern can appear under several tags).
- Flat frontmatter only; exemplars live in the body.
- Filename `{kebab-title}.md` IS the identifier (no `id` field).
- `related:` not validated in v1 (deferred).
- `INDEX.md` is generated, never hand-edited.
- Out of scope v1: automated triage tooling, periodic "librarian" passes, schema
  enforcement, custom site generators.
